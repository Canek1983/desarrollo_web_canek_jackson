# Tarea 1 - Desarrollo Web

## Descripción
En esta tarea se desarrolló una aplicación web utilizando HTML, CSS y JavaScript. El objetivo principal fue implementar un sistema básico que permite visualizar información en forma de tabla, así como la gestión de datos mediante formularios.

## Estructura del proyecto
El proyecto está compuesto por los siguientes archivos:
- `index.html`: contiene la estructura principal de la página web.
- `style.css`: define los estilos visuales, incluyendo el diseño de la tabla y formularios.
- `script.js`: implementa la lógica para la manipulación del DOM y la gestión de datos.

## Funcionalidades implementadas

### 1. Listado de datos
Se implementó un listado utilizando una tabla HTML (`<table>`), con encabezado (`<thead>`) estilizado en color azul claro para mejorar la legibilidad. Los datos se insertan dinámicamente mediante JavaScript.

### 2. Formulario de miembros
Se diseñó un formulario que permite registrar miembros mediante campos como:
- Nombre
- Edad
- Ciudad

Este formulario captura la información ingresada por el usuario y la agrega dinámicamente a la tabla.

### 3. Formulario de actividades
Se implementó un segundo formulario orientado a registrar actividades, incluyendo:
- Nombre de la actividad
- Fecha
- Descripción

Los datos ingresados también son procesados mediante JavaScript y pueden ser visualizados o gestionados dentro de la aplicación.

## Desarrollo
Para el desarrollo se utilizaron funciones en JavaScript que permiten:
- Capturar eventos de formularios (`submit`)
- Validar datos básicos
- Crear elementos dinámicos en el DOM
- Insertar información en la tabla

Se aplicó una separación clara entre estructura (HTML), estilo (CSS) y lógica (JavaScript).

## Decisiones de diseño
- Uso de tablas para representar datos estructurados.
- Implementación de formularios para la interacción del usuario.
- Aplicación de estilos simples para mantener claridad visual.
- Uso de JavaScript para manejo dinámico de datos.

## Consideraciones
El proyecto fue desarrollado con un enfoque académico, priorizando la claridad, organización del código y cumplimiento de los requerimientos. Se buscó una implementación funcional y comprensible.
